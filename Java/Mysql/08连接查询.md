## 连接查

询

​	在实际开发中，一般一个业务会对应多张表，因为将所有数据存在同一张表中会导致数据的冗余。

### 连接查询的种类

+ 语法出现的年代
  + SQL92
  + SQL99
+ 表的连接方式
  + 内连接
    + 等值连接
    + 非等值连接
    + 自连接
  + 外连接
    + 左外连接
    + 右外连接
  + 全连接（少用）

### 内连接

#### 等值连接

​	连接的条件是等量关系。

+ 找出每一个员工的部门名称，显示员工名和部门名。

```mysql
# 以下为 SQL99 语法
mysql> select
    ->  e.ename, d.dname
    -> from
    ->  emp e
    -> inner join # inner 可以省略
    ->  dept d
    -> on
    ->  e.deptno = d.deptno; # 在 on 的后面加连接的条件
    # 在后面可以继续写 where 条件
+--------+------------+
| ename  | dname      |
+--------+------------+
| SMITH  | RESEARCH   |
| ALLEN  | SALES      |
| WARD   | SALES      |
| JONES  | RESEARCH   |
| MARTIN | SALES      |
| BLAKE  | SALES      |
| CLARK  | ACCOUNTING |
| SCOTT  | RESEARCH   |
| KING   | ACCOUNTING |
| TURNER | SALES      |
| ADAMS  | RESEARCH   |
| JAMES  | SALES      |
| FORD   | RESEARCH   |
| MILLER | ACCOUNTING |
+--------+------------+
14 rows in set (0.00 sec)
```

+ 相比 SQL92 语法，SQL99 的语法结构更清晰，表的连接条件与 `where` 条件分离。

### 外连接

​	进行外连接的两张表，有主副之分，主要查询主表中的数据。如果副表中的数据没有和主表匹配，副表会自动模拟出 `NULL` 与之匹配。

​	如果是内连接，两张表是平等的，如果存在一个数据在两表间无法匹配，则不会查询出该数据。使用外连接可以保留主表的数据。

#### 左外连接

​	即左边的表是主表。

+ 找出所有员工的领导
  + 所有员工必需查询出来，哪怕没有上级领导。
  + 不希望丢失没有上级领导的那条员工数据。
    + 如果使用内连接，没有上级领导就不会显示。

```mysql
mysql> select
    ->  e.ename emp, m.ename mgr
    -> from
    ->  emp e
    -> left join
    ->  emp m
    -> on
    ->  e.mgr = m.empno
    -> order by
    ->  m.ename;
+--------+-------+
| emp    | mgr   |
+--------+-------+
| KING   | NULL  |
| ALLEN  | BLAKE |
| WARD   | BLAKE |
| MARTIN | BLAKE |
| TURNER | BLAKE |
| JAMES  | BLAKE |
| MILLER | CLARK |
| SMITH  | FORD  |
| SCOTT  | JONES |
| FORD   | JONES |
| JONES  | KING  |
| BLAKE  | KING  |
```

#### 右外连接

​	即右边的表是主表。

+ 可以将上面的左连接改为右连接写法

#### 全连接

​	两张表都是主表，即希望两张表的数据都不丢失。MySQL 不直接支持全连接。

### `union`

`union` 可以将查询的结果相加。可以认为 `join` 是表的横向拼接，`union` 是表的纵向拼接。

+ 找出工作岗位是 SALESMAN 和 MANAGER 的员工。

```mysql
mysql> select ename, job from emp where job = "SALESMAN"
    -> union
    -> select ename, job from emp where job = "MANAGER";
+--------+----------+
| ename  | job      |
+--------+----------+
| ALLEN  | SALESMAN |
| WARD   | SALESMAN |
| MARTIN | SALESMAN |
| TURNER | SALESMAN |
| JONES  | MANAGER  |
| BLAKE  | MANAGER  |
| CLARK  | MANAGER  |
+--------+----------+
7 rows in set (0.00 sec)
```
